# Outreach Pack

## Positioning

This is a tailored handover workflow concept for Office Tech Suite outreach. It should be presented as a discussion aid, not as finished technical, cyber, or security assurance.

Suggested first offer:

> a documentation and service-transition pilot that turns messy managed service provider notes into a reviewed handover pack, missing-information list, engineer checklist, runbook draft, and internal client-update draft.

The automation mapper can be shown when the conversation moves toward process improvement or repeated manual work.

## Pilot Ask

Ask for a 20-minute conversation about one low-risk documentation/service-transition pilot.

Suggested ask:

> Would it be worth a 20-minute call to see whether a lightweight documentation and handover workflow could reduce admin around one onboarding, migration, or service-transition pack?

## Pilot Deliverables

- One sample handover pack from dummy or sanitised notes.
- One missing-information checklist for engineer review.
- One short runbook or SOP draft.
- One internal client-update draft with neutral signoff placeholders.
- One before/after summary showing where admin time or review risk was reduced.
- Optional: one automation-opportunity map if they specifically want the automation angle.

## Boundaries

- No real client data in the first demo.
- No live AI/API calls in the first demo.
- No Microsoft Graph, PSA, SharePoint, Hudu, IT Glue, or Power Automate integration.
- No claim that outputs are technically correct without Office Tech Suite engineer review.
- No direct client-facing use unless Office Tech Suite approves the wording, data handling, and signoff.

## Access And Security Posture

- Start from dummy or sanitised notes only.
- Keep the prototype local and inspectable.
- Do not store notes in localStorage, sessionStorage, IndexedDB, or a backend database.
- Do not add analytics, telemetry, or external scripts beyond explicitly reviewed assets.
- If a live AI version is considered later, agree provider, data-retention terms, access controls, redaction, logging, and approval process before using real client data.

## Simple Price Shape

Use this as a discussion anchor, not a final quote:

- Discovery and sample pack: fixed-fee pilot.
- Repeatable handover packs: per-pack or day-rate model.
- Automation workshop: separate follow-on only if the first conversation shows appetite.

## Demo Script

1. Open the handover assistant on the default sample.
2. Show the client snapshot, key risks, and missing information.
3. Show the engineer checklist and runbook draft.
4. Export Markdown as the sample handover artifact.
5. Mention the automation mapper if the conversation moves toward process improvement.
6. Close by asking for a 20-minute call about one documentation/service-transition pilot.
