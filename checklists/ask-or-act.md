# Ask Or Act Checklist

## Ask First

- Would a wrong assumption change product direction?
- Could this affect production data, auth, billing, permissions, or deployment?
- Is a major dependency being added?
- Is there a destructive operation?
- Are there two valid approaches with different long-term costs?
- Is the user asking for product judgment rather than code execution?

## Act Without Asking

- Read files.
- Search code.
- Run non-destructive commands.
- Follow established project patterns.
- Fix obvious local bugs.
- Add focused tests for changed behavior.
- Format code using project tools.

## Question Quality

Ask one question at a time. Make it specific. Ask the question that unlocks the most progress.
